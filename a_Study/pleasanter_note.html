  • 手順を進めていく
  • ※「SQL Server 2019 Express with Advanced Servicesのインストール」セクションについて 「4. 「インストール ルール」の画面で「次へ(N)」をクリックしてください。」実行直後に、 マニュアルにはない、「インストールの種類」という画面が現れた。 →今のところ、SQL をインストールした記憶はないので、2 つの選択肢のうち、上段の、 「SQL Server 2019 の新規インストールを実行する(P)」を選択して、作業を進めることにする
  • 本日はここまで

  • ※「SQL Server 2019 Express with Advanced Servicesのインストール」セクションについて 「6, 「既定のインスタンス(D)」にチェックし「次へ(N)>」ボタンをクリックしてください。」実行直後に、 エラーダイアログが発生。 →「インスタンス名'MSSQLSERVER'は既に使用されています。続行するには一意のインスタンス名を指定してください」 →新規インスタンス作成に向けて、インスタンス名はデフォルトのまま作業を進めた結果、こうなった →すでに、以前作成した、デフォルト名のインスタンスが存在する可能性アリ。 →新規インストールではなく、既存のSQL サーバがあるものとして作業を進めてみる
  • 「インストールの種類」画面にて、「既存のSQL Server 2019 インスタンスに機能を追加する(A)」を選択し、 「次へ(N)」をクリックする(インスタンス名は、「MSSQLSERVER」)
  • 「機能の選択」画面にて、「データベースエンジンサービス」はグレースケールで、すでにチェックが入っていた スクロールすると、「共有機能」の「SQL クライアント接続SDK」にもグレースケールでチェックが入っていた →これは変更不可能なので、そのまま放置して「次へ(N)」をクリックする。
  • 参考ページの、「インスタンスの構成」ページは、実際には省略された。 →参考ページは、新規にインスタンスを作成する場合向けのものだからだと考えられる。
  • 「サーバと構成」画面にて、「次へ(N)」をクリックすると、プログレスバーが出てきて、待機。 →「データベース　エンジンの構成」は省略された。パスワード等を設定する画面。
  • 「SQL Server 2019 Express with Advanced Servicesの設定」セクションに進む →参考ページと、実際の表示が違うので、さかのぼって、既存のSQL を削除し、新規のSQL をインストールするルートを試す。
  • 「インストールの種類」画面にて、「SQL Server 2019 の新規インストールを実行する(P)」を選択して進む
  • 「ライセンス条項」にてチェックマークを入れ、進む
  • 「機能の選択」にて、スクロールしたところの「共有機能」のチェックが入った部分はそのままにして作業を進める
  • やはり、SQL Server の名前を決める部分で、インスタンス名のところで作業が止まってしまう。 →以下ディレクトリのファイルを削除して、初めからやり直すことを検討する。 →　C:\Program Files\Microsoft SQL Server →削除できなかった。「設定」から、アンインストールを実行した。 →所定のインスタンスだけを削除した。
  • 最初から、やり直す。
  • 「SQL Server 2019 Express with Advanced Servicesのインストール」を進めてゆく
  • 「データベース　エンジンの構成」にて、パスワードを設定する →パスワード＠＠＠　19960701pleasanter
  • 「SQL Server 2019 Express with Advanced Servicesの設定」セクションにて 参考サイトと実際の画面の構成が異なるので、飛ばした部分がある →「1. スタートボタンをクリック～」、「2. 左ペインにて～」
  • これで、「SQL Server 2019 Expressのインストール及び設定 」は終了
  • 「SQL Server Management Studioのインストール」に進む
  • 「1. 以下の～」に示されているのはバージョン17.X で、 リンク先のサイトには18.10 とある。ざっと17.X で検索してみたが見当たらないので、 18.10 で進めてみる。
  • 「SQL Server Management Studioのインストール」 →ロケーション　C:\Program Files (x86)\Microsoft SQL Server Management Studio 18
